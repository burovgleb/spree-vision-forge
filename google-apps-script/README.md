# Настройка заявок: Google Sheets + Telegram

Форма на главной и заявки из карточек товаров отправляются в Google Apps Script. Скрипт добавляет
строку в Google Sheets, отправляет сообщение через Telegram Bot API и записывает результат отправки
в столбец `Статус Telegram`. Для карточки отдельно сохраняются товар, артикул и стоимость.

## Что и где хранится

| Значение                 | Где указать                                        | Секрет                                  |
| ------------------------ | -------------------------------------------------- | --------------------------------------- |
| `SPREADSHEET_ID`         | Apps Script → Project Settings → Script Properties | Нет, но не публикуйте без необходимости |
| `SHEET_NAME`             | Там же; необязательно, по умолчанию `Заявки`       | Нет                                     |
| `TELEGRAM_BOT_TOKEN`     | Там же                                             | **Да**                                  |
| `TELEGRAM_CHAT_ID`       | Там же                                             | Условно                                 |
| `TELEGRAM_THREAD_ID`     | Там же; только для темы Telegram                   | Нет                                     |
| `VITE_GOOGLE_SCRIPT_URL` | `.env` локально или GitHub Actions Variable        | Нет, URL неизбежно виден браузеру       |

Токен Telegram нельзя помещать в `.env` сайта, React-код или GitHub Actions Variable: все значения
с префиксом `VITE_` встраиваются в публичный JavaScript. Токен должен быть только в Script
Properties самого Apps Script.

## 1. Создать Google-таблицу

1. Создайте новую таблицу в [Google Sheets](https://sheets.google.com/).
2. Скопируйте её ID из адреса. Для URL
   `https://docs.google.com/spreadsheets/d/1AbC_example_ID/edit#gid=0` ID равен
   `1AbC_example_ID`.
3. Заголовки вручную создавать не нужно: функция `setup` создаст лист и первую строку.

Таблица должна принадлежать тому же Google-аккаунту, который будет владельцем и исполнителем
Apps Script, либо быть доступна ему на редактирование.

## 2. Создать Telegram-бота и получить токен

1. Откройте официального бота [@BotFather](https://t.me/BotFather).
2. Отправьте `/newbot`, задайте имя и username бота.
3. BotFather пришлёт токен вида `123456789:AA...`. Это значение для
   `TELEGRAM_BOT_TOKEN`. Не добавляйте его в git и никому не пересылайте.
4. Откройте созданного бота, нажмите **Start** и отправьте ему любое сообщение. Бот не может первым
   начать личный диалог, поэтому этот шаг обязателен.

Если уведомления должны приходить в группу, добавьте бота в группу и отправьте там команду
`/start@username_бота` или другое сообщение, адресованное боту.

## 3. Создать Apps Script и внести закрытые настройки

1. Откройте [script.google.com](https://script.google.com/) → **New project**.
2. Назовите проект, например `Cache feedback`.
3. Откройте `Code.gs`, удалите шаблон и целиком вставьте содержимое файла
   [`Code.gs`](./Code.gs).
4. Нажмите **Project Settings** (шестерёнка слева). Рекомендуется поставить часовой пояс
   `Europe/Moscow`.
5. В разделе **Script Properties** добавьте:

   | Property             | Value                         |
   | -------------------- | ----------------------------- |
   | `SPREADSHEET_ID`     | ID из шага 1, без полного URL |
   | `SHEET_NAME`         | `Заявки`                      |
   | `TELEGRAM_BOT_TOKEN` | токен от BotFather            |

6. Сохраните свойства.

### Получить `TELEGRAM_CHAT_ID`

1. В редакторе Apps Script выберите в списке функций `showTelegramUpdates`.
2. Нажмите **Run**. При первом запуске разрешите скрипту внешние запросы.
3. Откройте **Execution log** внизу. Найдите последний объект `result`, затем
   `message.chat.id`. Это и есть `TELEGRAM_CHAT_ID`.
4. Добавьте найденное значение в Script Properties и сохраните.

Для личного чата это обычно положительное число. Для группы или супергруппы ID обычно
отрицательный, часто начинается с `-100`; минус является частью значения.

Если используется тема (topic) в форум-группе, возьмите `message.message_thread_id` из того же
ответа и добавьте как необязательный `TELEGRAM_THREAD_ID`. Для обычного личного или группового
чата это свойство не создавайте.

Если лог содержит пустой `result`, снова отправьте сообщение боту и сразу повторите запуск. Метод
`getUpdates` не работает, если этому боту ранее был настроен webhook; у нового бота webhook нет.

## 4. Авторизовать и проверить интеграцию

1. Выберите функцию `setup` → **Run**.
2. Google запросит права на таблицы и внешние HTTP-запросы. Выберите аккаунт владельца таблицы и
   разрешите доступ. Для собственного непубличного скрипта Google иногда показывает экран о
   непроверенном приложении: откройте дополнительные настройки и продолжите только если проект
   создан вами и код совпадает с [`Code.gs`](./Code.gs).
3. После успешного `setup` в таблице появится лист `Заявки` с заголовками.
4. Выберите `testIntegration` → **Run**.
5. Убедитесь, что в таблице появилась тестовая строка, в Telegram пришло сообщение, а в последнем
   столбце Telegram-статуса записано `Отправлено`.
6. Выберите `testProductIntegration` → **Run** и проверьте, что в строке и Telegram появились
   `Base · Грейдж · размер S`, артикул и стоимость. Тестовые строки после этого можно удалить.

Если лист уже был создан старой версией скрипта, `setup` автоматически добавит справа колонки
`Источник`, `Товар`, `Артикул` и `Стоимость`, не удаляя существующие заявки.

## 5. Опубликовать Apps Script как Web app

1. Нажмите **Deploy** → **New deployment**.
2. В **Select type** выберите **Web app**.
3. Укажите:
   - **Execute as:** `Me`;
   - **Who has access:** `Anyone`.
4. Нажмите **Deploy** и скопируйте URL, который заканчивается на `/exec`.

Откройте скопированный URL в новой вкладке. Исправный deployment покажет
`{"ok":true,"service":"cache-feedback"}`. Это проверяет доступ к Web app; запись в таблицу и
Telegram уже проверены функцией `testIntegration`.

Не используйте адрес `/dev`: он доступен только редакторам скрипта. Если корпоративная политика
Google Workspace не предлагает доступ `Anyone`, публикацию должен разрешить администратор либо
скрипт нужно развернуть из аккаунта, где анонимный Web app разрешён.

После любых изменений `Code.gs` одного сохранения недостаточно: откройте **Deploy → Manage
deployments**, нажмите карандаш, выберите **New version** и снова **Deploy**. URL `/exec` при этом
остаётся прежним.

## 6. Подключить URL к сайту

### Локальный запуск

1. Скопируйте `.env.example` в `.env`.
2. Вставьте URL `/exec`:

   ```dotenv
   VITE_GOOGLE_SCRIPT_URL=https://script.google.com/macros/s/ВАШ_DEPLOYMENT_ID/exec
   ```

3. Перезапустите dev-сервер: Vite читает `.env` при старте.

Файл `.env` уже исключён из git. Сам URL не является секретом, но токен Telegram в этот файл всё
равно помещать нельзя.

### Текущий деплой через GitHub Pages

Workflow уже передаёт `VITE_GOOGLE_SCRIPT_URL` из GitHub Actions Variables. В репозитории откройте:

**Settings → Secrets and variables → Actions → Variables → New repository variable**

Создайте переменную:

```text
Name:  VITE_GOOGLE_SCRIPT_URL
Value: https://script.google.com/macros/s/ВАШ_DEPLOYMENT_ID/exec
```

После сохранения запустите **Actions → Deploy Pages → Run workflow** либо отправьте изменение в
`main`. URL публичный, поэтому здесь нужна именно **Variable**, а не Secret.

Для другого хостинга создайте build-time environment variable с тем же именем и заново соберите
сайт.

## 7. Финальная проверка

1. Откройте опубликованный сайт в приватном окне.
2. Заполните имя, контакт, один-два интереса и комментарий.
3. Нажмите **Отправить заявку** только один раз — кнопка должна показать `Отправляем…`, затем
   экран благодарности.
4. Проверьте новую строку в Google Sheets и сообщение в нужном Telegram-чате.
5. Убедитесь, что `Статус Telegram` равен `Отправлено`.
6. Откройте `/catalog`, выберите модель, нажмите **Оставить заявку** и повторите проверку. В
   уведомлении и отдельных колонках таблицы должны быть выбранные модель, размер, артикул и цена.

## Диагностика

- **Сайт сообщает, что форма не настроена:** `VITE_GOOGLE_SCRIPT_URL` отсутствовал во время
  сборки, имеет адрес `/dev` или не похож на URL `script.google.com/macros/s/.../exec`.
- **Экран благодарности есть, а строки нет:** откройте Apps Script → **Executions**. Браузер
  отправляет запрос в режиме `no-cors`, поэтому из статического сайта нельзя прочитать внутреннюю
  ошибку Apps Script. Чаще всего причина — доступ Web app не `Anyone`, неверный `SPREADSHEET_ID`
  или после правки скрипта не опубликована новая версия.
- **Строка есть, Telegram не пришёл:** причина записана в `Статус Telegram`. `chat not found`
  означает неверный `TELEGRAM_CHAT_ID` или пользователь ещё не нажал Start; `bot was blocked`
  означает, что бот заблокирован; для группы проверьте, что бот не удалён.
- **`showTelegramUpdates` возвращает пустой список:** сначала отправьте боту новое сообщение. Если
  бот уже использовался другой системой с webhook, создайте отдельного бота для сайта либо удалите
  webhook осознанно.
- **После правки Code.gs поведение не изменилось:** опубликуйте **New version** существующего
  deployment.
- **Подозрение на утечку токена:** немедленно отзовите/перевыпустите токен через BotFather и
  замените `TELEGRAM_BOT_TOKEN` в Script Properties.

Endpoint Apps Script публичный по природе, иначе анонимный посетитель сайта не сможет отправить
форму. В реализации есть honeypot, серверная проверка/ограничение длины полей, защита таблицы от
формул, блокировка параллельной записи, подавление повторов по ID и три попытки обращения к
Telegram. Для сайта с заметным объёмом спама следующим уровнем защиты будет CAPTCHA или
собственный серверный proxy.
